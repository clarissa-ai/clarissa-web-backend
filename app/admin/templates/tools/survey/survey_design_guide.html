Survey Basics

Survey Design
Surveys must be designed as DAGs (Directed Acyclic Graphs). There must be a root (initial) question that is presented to the user, and every option must have a "next question", there the user will be directed to after selecting that option.
Survey Elements
Surveys contain 3 main elements: the title page, question pages, and the final summary page.

Title page:
The title page is the first page a user will see when taking the survey. It has a title (same as survey title), description, and set of "links" (resources for the user). Each link/resources has a title, description, and a URL that the resource redirects to when clicked.

Question pages:
Each question page will have a title, a description of the question, and a set of options the user can choose. Each option has a "next question" field, that determines what question the user will be redirected to if they choose that option. Question options can also direct to a final summary page.

Final Summary pages:
The Final Summaries represent the last pages that a user can get to in a survey. Final summaries have a title, a short description, and "information groups". Information groups are resources for the user, containing a title, a list of details, and a link that the resource redirects to.
Publishing surveys
A survey has 2 fields determining whether or not the survey is published. "Main" and "Active" represent whether or not the survey is the main survey, displayed on the home page, and whether or not the survey is part of the active surveys, displayed to users in the surveys portion of the application.

Questions

Question Core Elements
Title: The title of each question
Description: A description that gives the survey-taker a better understanding of the objective of the survey and what kind of information they can get from it.
Default next:
The survey designer can provide a default "next question" that a survey taker will be directed to after this question. The default next can also be set to send the user to the summary page.
Question type:
The survey designer may chose from several different question types, that change how the user can input their response to the question. The current question types are: multiple select (user can select multiple options from a list), single select (user can only chose one option from a list, max 7 options), dropdown (same as single select except allows for more options), and short answer (text area input).
Options:
For certain types of questions, the user is presented a set of options that they can chose from. When adding an option the survey designer can also add 3 additional details that are used to determine what the user is presented after selecting that option:
  • Next question: The question the user is directed to after selecting this option. If no next question is selected, the option will default to using current question's "default next" question. In a multiple select question, all the user's selected options are tallied, and the user is directed to the most frequent next question.
  • Associated Summary: The associated summary represents the summary that the user should be led to for selecting that option. For example, in a question where the user is presented a set of symptoms, where different symptoms are associated with different conditions, you can make summaries for each of the conditions and set those options to be associated with the summaries for those conditions.

Summaries

Basic Elements:
Title: The title of the summary is the final result that the user sees after completing the survey. It should be intentionally designed to be a culmination of the options the user selected.
Description: A short quote that supplements the title (usually only a few words). Some examples could be "Recommendation", "Healthy Tips", and "Urgent".
Info Groups:
Info groups are groups of information presented to the user that give the user more details to act on regarding the results of their survey. Info groups have a title, a link (the title is a hyperlink), and a group of details (text bullet points supplementing the title of the info group).
